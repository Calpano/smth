Give your agent its own (phantom) browser.

smth is a local MCP server that lets your agent screenshot pages, inspect the DOM, check fonts and colors, and interact with forms — all in a persistent browser session.

Capabilities

Three modes. Sixteen tools.

See

Read any page without touching it. Get screenshots at any viewport, extract visible text as Markdown, inspect the full DOM through focused lenses, and analyze fonts, colors, and WCAG contrast ratios — all returned as structured data your agent can reason about.

browser_see_visual browser_read_text browser_see_dom browser_see_colors browser_see_color_pairs browser_see_fonts

Act

Drive the browser like a user. Click elements by id or CSS selector, type into form fields, hover to reveal tooltips and dropdown menus. Every interaction waits for navigation to settle before returning.

browser_launch browser_goto browser_click browser_hover browser_type

Compare

Understand what changed. Capture named DOM snapshots before and after any interaction, then diff them line by line. Or use fetch_dom_content to automatically strip shared chrome — nav, footer, sidebar — and return only the unique content of a page.

browser_remember_dom browser_dom_compare browser_doms fetch_dom_content
Get started

Up in
one command.

smth runs as a persistent local service. One docker compose up, then add four lines of JSON to any project.

  • No cloud. No telemetry. Runs entirely on your machine.
  • Persistent session — browser stays open between tool calls.
  • Works with Claude Code, Cursor, Cline, and any MCP client.
  • Chromium included — no separate install needed.
  • Or run directly in Node.js with your own Chrome.
bash
$ git clone https://github.com/Calpano/smth $ cd smth $ docker compose up -d
.mcp.json — project root
{ "mcpServers": { "smth": { "type": "http", "url": "http://localhost:3000/mcp" } } }

Reference

All 16 tools

ToolModeDescription
browser_launch action Start a session and open a URL. Call before any other tool.
browser_goto action Navigate to a new URL without relaunching the browser.
browser_click action Click an element by id or CSS selector.
browser_hover action Hover an element and return a before/after text diff.
browser_type action Type text into a form field. Clears first by default.
browser_see_visual read-onlyScreenshot — full page or viewport, any emulated device.
browser_read_text read-onlyPage text as Markdown + interactive element selectors.
browser_see_dom read-onlyLens-filtered compact HTML or text search with context lines.
browser_see_colors read-onlyAll computed colors with usage counts and element selectors.
browser_see_color_pairsread-onlyText/background pairs with WCAG 2.2 contrast ratios and AA/AAA flags.
browser_see_fonts read-onlyAll fonts grouped by family, size, and weight.
browser_remember_dom action Save a named DOM snapshot for later comparison.
browser_doms read-onlyList all saved snapshots with size, lens, and timestamp.
browser_dom_compare read-onlyDiff two snapshots, or subtract a background to isolate content.
fetch_dom_content action Open a URL, auto-strip shared chrome, return only unique content.
browser_list_devices read-onlyList device names for browser_see_visual emulation.
